## Issue Reported

User reports that Microsoft Outlook is generating send and receive errors and that Windows Update has not completed successfully in several weeks. The user has not made any intentional changes to the machine. Both Outlook and Windows Update are failing silently from the user's perspective with no clear error message describing the root cause. User is unable to receive email reliably and the machine is accumulating unpatched security updates.

---

## Investigation

**Free space check — File Explorer:**

1. Opened File Explorer → This PC on WIN11-NM-01.
2. Observed Local Disk C: storage bar — colour was red, indicating a critical low space state.
3. Recorded free space: **4.09 GB free of 63.0 GB**.
4. Root cause hypothesis formed: insufficient free space is blocking Outlook cache operations and Windows Update package extraction.

**Storage breakdown — Settings → System → Storage:**

5. Navigated to Settings → System → Storage.
6. Confirmed: **58.9 GB used** out of 63.0 GB total — **4.08 GB free**.
7. Reviewed storage category breakdown:

   | Category | Space Used |
   |---|---|
   | Temporary files | **30.0 GB** |
   | Installed apps | 8.35 GB |
   | Other people | 1.97 GB |
   | Documents | 1.00 GB |
   | Total used | 58.9 GB |

8. Temporary files at 30.0 GB confirmed as the dominant cause of the low space condition.
9. Storage Sense was shown as On — however the schedule had not triggered a cleanup before the drive reached the critical threshold.

**Disk Cleanup:**

10. Ran Disk Cleanup (cleanmgr) on drive C:.
11. Reviewed files available for deletion. DirectX Shader Cache: 36.0 KB — not selected. Delivery Optimization Files: 95.4 MB — not selected (retained for update rollback). Recycle Bin: 5.20 MB — selected. Temporary files: 0 bytes at this point, cleared via Storage Settings. Thumbnails: 11.0 MB — selected.
12. Total recoverable via Disk Cleanup: **23.0 MB**
13. Disk Cleanup confirmation presented — OK selected.

**Recycle Bin permanent deletion:**

14. Windows presented permanent delete confirmation dialog: "Are you sure you want to permanently delete these 2 items?"
15. Confirmed with user that no recently deleted files were needed for recovery.
16. Selected Yes — Recycle Bin cleared.

**Root Cause Confirmed:**

Accumulated temporary files (30.0 GB) generated by Windows, application installers, and update processes had consumed the system drive to a critical level. Storage Sense was enabled but had not triggered a cleanup cycle before the drive reached 4.08 GB free. At that threshold both Outlook and Windows Update cannot allocate the working space they require.

---

## Actions Taken

1. Recorded pre-cleanup baseline: 4.09 GB free of 63.0 GB (red storage bar in File Explorer).
2. Reviewed Storage Settings breakdown — identified 30.0 GB temporary files as primary target.
3. Navigated to Settings → System → Storage → Temporary files → removed temporary files.
4. Ran Disk Cleanup (cleanmgr) on C: — selected Recycle Bin (5.20 MB) and Thumbnails (11.0 MB).
5. Confirmed permanent Recycle Bin deletion via dialog — user confirmed no recovery needed.
6. Clicked OK in Disk Cleanup — cleanup completed without error.
7. Returned to File Explorer → This PC and recorded post-cleanup free space: **34.5 GB free of 63.0 GB**.
8. Confirmed storage bar returned to normal (non-red) state.
9. Advised user to relaunch Outlook and retry send and receive.
10. Advised user to navigate to Settings → Windows Update and retry pending updates.

---

## User Communication

The user was informed that the root cause of both the Outlook failures and the Windows Update failures was the same: the system drive had accumulated 30 GB of temporary files over time, leaving only 4 GB of free space on a 63 GB drive. They were advised that this is not caused by anything they did wrong and is a normal maintenance task that should be performed periodically.

The user was also informed that:

Storage Sense is already enabled on their machine and will continue to attempt automatic cleanup on its schedule. They can also manually trigger a cleanup via Settings → System → Storage → Cleanup recommendations at any time. If Outlook or Windows Update errors return, they should check the free space value in File Explorer first before raising a new ticket. Any item sent to the Recycle Bin should be considered as pending deletion and not a permanent storage location.

## Preventive Recommendations

1. Configure Storage Sense to run every 14 days rather than the default monthly schedule on machines with drives smaller than 128 GB.
2. Set Storage Sense to automatically delete files in the Recycle Bin after 30 days.
3. If endpoint management is in use, add a low disk space alert at 10 GB remaining to allow proactive intervention before application failures occur.
4. Advise users to empty their Recycle Bin monthly as a simple maintenance habit.
5. Review whether the 63 GB drive capacity is appropriate for this user's role. If temporary file accumulation recurs within 30 days at this rate, a drive capacity review may be warranted.
